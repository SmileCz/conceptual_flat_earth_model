🚨 KRITICKÁ ANALÝZA: "Conceptual Flat Earth Model"

Narazil jsem na GitHub projekt, který prezentuje flat earth jako "conceptual model" s profesionálně vypadající vizualizací. Provedl jsem hlubokou technickou a vědeckou analýzu. Výsledky jsou alarmující. 🧵

═══════════════════════════════════════════

🔬 VĚDECKÁ MANIPULACE:

1️⃣ TERMINOLOGICKÝ KLAM
"Conceptual model" vytváří false legitimitu - jako bych řekl "conceptual perpetual motion machine". Projekt používá NASA data (DE405 ephemeris), VSOP87 planetary theory a Meeus algoritmů...

...ale vkládá je do geometrie, která nemůže vysvětlit:
❌ Časová pásma (nemohou všechna existovat na disku)
❌ Satelitní orbity (které projekt ironicky simuluje!)
❌ Gravitaci (kam by spadaly objekty?)
❌ Foucaultovo kyvadlo
❌ Cirkapolární hvězdné stopy

2️⃣ MATEMATICKÝ TRIK
Projekt tvrdí "geometrickou ekvivalenci" FE ↔ Globe:
"Dome height = Sphere radius, proto stejné elevace"

➡️ To je TAUTOLOGIE! Nastavil parametry tak, aby seděly - není to důkaz fyziky.
➡️ Cherry-picking: funguje jen pro úhly z 1 bodu, ignoruje horizontální vzdálenosti
➡️ Žádná gravitace v modelu (protože by vše spadlo na pól nebo atmosféra unikla)

3️⃣ SATELITY - VLASTNÍ GÓL 🛰️
Projekt simuluje ISS, Hubble, Starlink pomocí Keplerových orbit.
TLE data (Two-Line Elements) fungují JEN na sféře!

Logická past:
IF satelity obíhají (vidíme je)
AND projekt počítá pozice správně pomocí sférické fyziky
THEN Země MUSÍ být sféra 🌍

4️⃣ UKRADENÁ NASA DATA
44 solar eclipses (2021-2040) od Fred Espenak (NASA)
- Vypočítáno pro SFÉRICKOU Zemi
- S Newtonovou gravitací
- S relativistickými korekcemi

Projekt je vezme, přemapuje na FE a tvrdí "funguje na obou modelech".
❌ Saros cyklus vyžaduje 3D gravitaci Slunce-Země-Měsíc
❌ FE nemá mechanismus pro správný timing zatmění

💻 SOFTWAROVÁ ARCHITEKTURA - TECHNICKÝ ROZBOR:

Paradox: Sofistikovaná implementace špatné myšlenky

✅ POZITIVA:
- Modularizace (core/render/ui)
- Event-driven architecture
- 5 ephemeris pipelines
- WebGL rendering (Three.js)
- PWA support

🔴 KRITICKÁ PORUŠENÍ:

1️⃣ BUILD ARTIFACTS V GITU
Komentář tvrdí: "js-min/ is gitignored"
Reality: js-min/ NENÍ v .gitignore → tracknuto!
➡️ 100+ souborů v každém commitu
➡️ Merge hell
➡️ AMATÉRSKÁ CHYBA (junior level)

2️⃣ SINGLE-BRANCH WORKFLOW
❌ Žádné feature branches
❌ Žádné pull requests / code review
❌ Žádný CI/CD (.github/workflows/ neexistuje)

Profesionální projekty:
✅ React: min 2 reviews, 30+ CI checks
✅ Vue: Git Flow s dev + main
✅ Three.js: Feature branches + Lighthouse CI

3️⃣ DRY VIOLATIONS
~200 řádků DUPLICITNÍHO kódu:
- projectStar() vs projectSatellite() - 90% stejné
- Sun/Moon/Planet projection - copy-paste 5×

➡️ Bug = oprava na 5 místech
➡️ 80% redundantního kódu

4️⃣ SOLID VIOLATIONS
app.js = 1755 ŘÁDKŮ s 9+ odpovědnostmi:
- State management
- Ephemeris calculations
- Transformations
- Projections
- Cache
- ...

➡️ Single Responsibility Principle: ZNIČEN
➡️ Monolitní state objekt (100+ properties)

5️⃣ ZERO TESTŮ
❌ Žádné unit tests
❌ Žádné integration tests
❌ Žádný test coverage

Pro 1755řádkovou třídu s astronomickými výpočty je to NEUDRŽITELNÉ.

🔴 KRITICKÁ ZÁVAŽNOST:

- Eticky problematický (zneužití NASA dat)

❌ Vědecky nevalidní (ignoruje fyziku)
❌ Profesionálně neudržitelný (amatérské chyby)

⚠️ VAROVÁNÍ PRO UŽIVATELE:

✅ POUŽIJ RADĚJI:
